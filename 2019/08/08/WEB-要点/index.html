WEB 要点

Posted on 2019-08-08

攻防世界Web新手区解析

WEB1–view source

  • 主要考察浏览器快捷键F12的用法。

    WEB2–Get&Post

  • 主要是考察Get和Post不同的传参方法以及火狐上hackbar插件使用

    • Get的参数主要是通过URL后附加参数传递参数，有一定风险

      • 传递参数a=1，burpsuit拦截数据包

    • Post传递参数不会显现在页面上，较为安全

      • 传递参数b=2，burpsuit拦截数据包

    • Get与Post不同之处在于首先URL声明传参类型，其次Post方式含有Content-Type: application/x-www-form-urlencoded

      WEB3–Robots

      Robots是什么

dirsearch

  • 使用脚本dirsearch来搜寻目标网站下的文件
    • 命令行：python3 dirsearch.py -u URL -e php
  • 本题可以直接在URL后加/robots.txt，一般网站都会如此配置

    WEB4–backup

  • 同第三题，用dirsearch脚本扫描该url，发现index.php.bak，下载即可

    WEB5–cookie

    cookie

  • cookie主要是用户打开网页时的命令标识符，它代表着当前用户的信息以区别其他用户。

    解析

  • Firfox浏览器F12打开页面调试，点击存储->cookie发现有一个look here的数据包含了cookie.php文件名。
  • 打开URL/cookie.php，显示查看response。F12->网络第一个URL即可。
  • 也可利用burpsuit抓包查看页面回复。

    WEB6–disabled_button

  • F12查看页面调试，html文件发现input属性含有disabled=””，将次删去点击按钮跳出flag

    burpsuit抓包

    提交前

提交后

  • 可以利用input的属性中添加auth=flag，并利用post方式提交即可

WEB7–simple_js

  • 审计html代码发现问题

  • 无论输入任何值，都会输出FAUX PASSWORD HAHA,原因在于for循环p加载密码为p += String.fromCharCode((o = tab2[i]))，修改其为p += String.fromCharCode((o = tab[i]))

  • “\x35\x35\x2c\x35\x36\x2c\x35\x34\x2c\x37\x39\x2c\x31\x31\x35\x2c\x36\x39\x2c\x31\x31\x34\x2c\x31\x31\x36\x2c\x31\x30\x37\x2c\x34\x39\x2c\x35\x30” 为题目所包含的莫名数字，将其转换为十进制并输入到已修改的js中得到flag。

WEB8–xff_referer

X-Forwarded-For–xff

  • X-Forwarded-For简称xff，它代表客户端，也就是HTTP的请求端真实的IP，只有在通过了HTTP 代理或者负载均衡服务器时才会添加该项。

    Referer

  • HTTP Referer是header的一部分，当浏览器向web服务器发送请求的时候，一般会带上Referer，告诉服务器我是从哪个页面链接过来的

    burpsuit抓包并伪造XFF和Referer

WEB9–weak_auth

  • 简单的登录验证，并且没有验证码什么的可以直接爆破密码。

    burpsuit

  • 首先抓包，然后选择Intruder中的payloads，利用字典爆破该密码即可。

WEB10–webshell

  • 简单的按照题目所给的利用蚁剑连接上去，密码就是shell。

WEB11–command_execution

Web Application Firewall-WAF

  • WAF可以用来屏蔽常见的网站漏洞攻击，如SQL注入，XML注入、XSS等。一般针对的是应用层而非网络层的入侵，从技术角度应该称之为Web IPS。其防护重点是SQL注入。
  • 本题目对输入部分未加判断，发现可以利用ubuntu命令特性“;”发送 ;find / -name “*flag*“找到flag.txt，打印改文件内容即可。

    WEB12–simple_php

    题目要点

  • 通过查看php代码发现题目主要是考察php弱类型比较

    php弱类型

    ==与===

  • ==的两个操作数比较时会先将两个操作数转化成同一类型再做比较
    • ‘abc’==0 –>(‘abc’=0,0=0) –> true
    • ‘1abc’==0 –>(‘1abc’=1,1!=0) –>false
  • ===的两个操作数比较时会先比较操作数类型然后再比较操作数值
    • ‘abc’===0 –>(string!=int) –> false
    • 0===0 —> (int==int) –> (0=0) –> true

      is_numeric()

  • 如果is_numeric()函数参数是数字或者数字字符串该函数返回true
    • is_numeric(123) –> true
    • is_numeric(‘123’) –> true

      题解

  • URL?a=abc&b=12345a
