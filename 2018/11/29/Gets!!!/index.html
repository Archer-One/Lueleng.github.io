Only Gets Can Use!

  • 令人抓狂的rop

源码

  • 编译 gcc gets.c -o gets -fno-stack-protecter

分析

ROP流程

  • 想办法从got.plt上捞东西出来(base)
  • 把捞出来的东西加上offset(base+offset)
  • 放回memory再call它

从got.plt上取值

  • 使用pop_rsp_r13_r14_r15_ret,暂时把stack移动到got上，用pop把值拿出来
  • 在最后ret对应的格子上，预先填好leave_ret;并在一开始就先设好rbp准备好migration stack
  • 取出的base在r13上
  • got.plt在FULL RELRO下是唯读的

先把base转到rbx

  • call_libc_csu_init
    • 基本上是个什么都不做的函数
    • 但这个函数使用了rbx,r12-r15,所以会把值push在stack上
  • 对好位置接上pop rbx,就可以把原本在r13的值转移到rbx

取得base在中rbx加offset

  • add ebx esi； ret； = 01 f3 c3
  • 有mov cs:__bss_start,1就会有这个gadget
  • ebx=ebx+esi(offset)
  • 再用csu_init把ebx放回memory

补上High 4bytes Address

  • 前面的做法最后只有ebx(low part)被放到memory上
  • 再重做一次，去拿base的high part
    • stack可以不是8byte-aligned(对齐)，歪4byte去拿gets@got的high part
    • 最后 push rbx时也要歪，这样就可以接在之前push rbx的后4byte

Call System

  • 有call qword ptr [r12 + rbx*8],我们把rbx设为0，r12设置为system_addr,前面设置好rdi那么便能实现。
  • 梅大佬做法：设置好rdi后，那么leave回存放system_addr即可。

神坑

  • 其中有一个神坑
  • 当我们利用csu_init函数将存放在r13上的值存放到栈上，我们的做法是首先在值的前一个位置利用gets函数存放pop_rbx等操作
  • 当gets读取完之后会帮添加‘\n’(栈上变为\x00’)覆盖下面地址的最低位的一个byte，所以我们要人为的在pop_rbx首地址添加’\n’来防止接下来的值被修改
