Binary Exploit

动态编译(daynamically)

  • 只需要x.dll文件。
  • 在程序执行需要该动态链接库的地方加载x.dll。
  • 然后获取需要的x.dll库里面的函数或数据.

ASLR Protection

  • Address Space Layout Randomization
  • 每次函数库载入的位置(base address)会随机化
  • 避免被猜到
  • address = base +offset

Defeat ASLR

  • Libc的base可以从很多地方推得
  • base = address - offset
  • 利用.text内的函数来泄露资讯
  • 某些情况下的buffer overflow，或是有任意记忆体位置写入的漏洞，可以覆盖掉部分地址
  • 原本的值跟要改写的值，相差不多于0x10000可以只覆盖最低2bytes
  • ASLR以page(4K)为单位对齐，故最底下的1.5byte值是确定的，只要赌0.5byte=1/16的几率

DynELF

  • 每次往前移动一个page，检查开题是否恰好是ELF的magic number，找出libc的base address
  • 根据libc里的header和hash table，找出symbol

源码

  • 生成运行栈溢出，32位的elf文件
  • 开启了ASLR

分析

  • 因为这次是动态编译而非静态编译，所以没有足够的gadget来组成System Call，我们选择ret回libc中的System函数
  • 但是System的地址需要我们去计算所得
  • address=base+offset

步骤

1.首先通过ldd和readlf命令查看导入的libc库，以及该库中有哪些函数

2.因为可以利用栈溢出所以我们先找到offset=116

3.我们通过ret到put_plt位置并设置好栈上的数值，首先跳到put_plt,这时我们先把A_addr入栈那么他就会把A_ddr当做一会要返回的地址，然后入栈plt_gots作为put_plt第一条跳转的语句，并泄露出puts在libc中的address，那么我们只要用泄露出的puts_add减puts_offset得base

4.利用base+system_offset=system_address，然后我们重新执行A函数利用rop跳转到gets函数获得第二次输入机会，并将gets函数的返回地址改为System_address，第二次输入时我们需要有一个可读并可以写的地方，因为是动态导入，那么puts函数是在运行时导入，put_got地址上的函数一定是运行时写进去然后读取跳转，那么我们就可以把这个当做写入的地址，并把次当做System的参数。另外这个时候offset已经变了但是我们可以利用slide ret来跳转。

Stack Migration

  • Gadget: leave; ret;
  • leave = mov esp,ebp; pop ebp;

Search Specific Symbols

  • 在远端libc版本不明的情况下，不知道各个function的offset
  • 没有办法回推base
  • 半有办法找到特定的function
  • 解法
    • Dump整个libc回来
    • 仿照_dl_fixup,利用ELF header里的资讯直接找

GOT Hijacking

  • 通常搭配任意位置写入漏洞来使用
    • Format String Vulnerablity
    • Heap Exploitation

任意地址读取

  • 若字符串位于堆栈上，可以利用%S读取记忆体中任意位置的内容
  • 构造适当的输入，将要读取的位置作为参数传递给%s，皆可以作为字串印出目标的内容

任意地址写入

  • 如格式字串位于堆栈上，可以利用%n对记忆体中任意位置写入资料
  • 类似任意读取时的做法，一样可以将要写入的目标位置放在堆栈上
  • 利用%123c这样的格式字串控制输出的长度
  • 在使用%hhn可以写入一个byte到目标位置

简单formatstr

格式化字符串

%[parameter][flags][field width][.precision][length]type]

  • parameter
    • n$, 获取格式化字符串中的指定参数
  • flag
  • field width
    • 输出的最小宽度
  • precision
    • 输出的最大长度
  • length 输出的长度
    • hh，输出一个字节
    • h，输出一个双字节
  • type
    • d/i，有符号整数
    • u，无符号整数
    • x/X，16进制的unsigned int。x使用小写字母；X使用大写字母。如果指定了精度，则输出的数字不足时左侧补0.默认精度为1.精度为0且值为0，则输出为空。
    • o，8 进制 unsigned int 。如果指定了精度，则输出的数字不足时在左侧补 0。默认精度为 1。精度为 0 且值为 0，则输出为空。
    • s，如果没有用l标志，输出null结尾字符串知道精度规定的上限；如果没有指定精度，则输出所有字节。如果用了l标志，则输出函数参数指向wchar_t型的数组，输出是把每个宽字符转化为多字节字符，相当于调用wcrtomb函数。
    • c，如果没有用 l 标志，把 int 参数转为 unsigned char 型输出；如果用了 l 标志，把 wint_t 参数转为包含两个元素的 wchart_t 数组，其中第一个元素包含要输出的字符，第二个元素为 null 宽字符。
    • p，void*型，输出对应变量的值。printf(“%p”,a)用地址的格式打印变量a的值，printf(“%p”,&a)打印变量a所在的地址。
    • n，不输出字符，但是把已经成功输出的字符个数写入对应的整形指针参数所指的变量。
    • %,’%’字面值，不接受任何flags，width

      小技巧

  • 利用%x来获取对应栈的内存，但建议使用%p，可以不用考虑位数的区别。
  • 利用%s来获取变量所对应地址的内容，只不过有零截断。
  • 利用%order￥x来获取指定参数的值，利用%order￥s来获取指定参数对应地址的内容。
