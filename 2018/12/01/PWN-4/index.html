Format String Vulnerablity

Format String

  • 在输出或字串处理函数中用俩表示输出的字串格式

  • 错误的使用方式，直接将使用者的输入作为fmt使用

  • 原本想要的是printf(“%s”,str),但不小心写成gets(str);printf(str);
  • 这个错误不容易发现，因为如果没有测试特殊的输入，执行结果会跟预期的一样
    +现在大部分的编译器会有warning了

Leak libc base

  • 程序执行时的起点是_start，接下来会把main作为参数传入_libc_start_main
  • _libc_start_main会先完成一些初始化操作，之后才会call main

  • 总之main的return address会指向libc内部，且存放在stack上的值可用%x(64位用%lx)泄露出来

  • 32位用的是 %x

64-bit

  • 64位用的是 %lx
    • 64位下前六个参数会在寄存器上
    • demo1 不知道如何泄露main的地址
    • LD_LIBRARY_PATH=.

      64-bit时null byte 问题

  • printf的format string 遇到\0就会结束掉，而不会再往下解
  • 64-bit address高位必定是0，所以address不可以摆在最前面
  • Buffer在fmt后再接address是可以的，但要小心参数的位置不要算错。如果fmt部分不做padding的话，后半address的位置会随前半fmt的长度改变
